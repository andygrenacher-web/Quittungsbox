# Quittungsbox

Mobile-first installable Android app for Swiss German users to photograph receipts, run OCR, generate PDF, and archive locally. No forms, no categories, no backend, no cloud.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 5000)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)

## Quittungsbox stack

- React + Vite + Wouter + Tailwind (web layer)
- Capacitor 8 (Android APK shell)
- `@capacitor-mlkit/document-scanner` — ML Kit Document Scanner (native camera + edge detection + perspective correction)
- `@capacitor/camera` — fallback camera if ML Kit unavailable
- `@capacitor/filesystem` — save PDFs to `Documents/Quittungsbox/`
- `@capacitor/preferences` — store receipt metadata
- Tesseract.js — local OCR on corrected images (no cloud)
- jsPDF — PDF generation

## Building the Android APK

```bash
# 1. Build web layer
pnpm --filter @workspace/quittungsbox run build:android

# 2. Sync into Android project
cd artifacts/quittungsbox && npx cap sync android

# 3. Build APK (requires Android SDK + Java 17)
cd artifacts/quittungsbox/android && ./gradlew assembleDebug
```

APK output: `artifacts/quittungsbox/android/app/build/outputs/apk/debug/app-debug.apk`

GitHub Actions workflow at `.github/workflows/build-android-apk.yml` does this automatically on every push to `main` that touches `artifacts/quittungsbox/`.

## Where things live

- `artifacts/quittungsbox/` — web app + Capacitor project
- `artifacts/quittungsbox/src/lib/capture.ts` — camera capture (ML Kit Document Scanner → Camera fallback)
- `artifacts/quittungsbox/src/lib/scanner.ts` — canvas-based document detection + image filters
- `artifacts/quittungsbox/src/lib/ocr.ts` — Tesseract OCR + filename builder
- `artifacts/quittungsbox/src/lib/storage.ts` — Filesystem (Android) / IndexedDB (web)
- `artifacts/quittungsbox/src/lib/pdf.ts` — PDF generation
- `artifacts/quittungsbox/capacitor.config.ts` — Capacitor config (appId: ch.quittungsbox.app)
- `artifacts/quittungsbox/android/` — generated Android project (committed, build artefacts gitignored)

## Architecture decisions

- **ML Kit Document Scanner, not text recognition**: `@capacitor-mlkit/text-recognition` doesn't exist as a free npm package. The Document Scanner plugin provides excellent native camera + edge detection + perspective correction. Tesseract then runs on clean, well-cropped images — significantly better than raw photos.
- **Tesseract.js only for OCR**: No cloud OCR. Local-only is a hard requirement.
- **IndexedDB on web, Filesystem on Android**: `isNative()` gates all Capacitor calls so the web preview always works without Android SDK.
- **`alreadyCorrected` flag on CaptureResult**: When ML Kit Document Scanner returns an image, `scanImage()` is skipped (perspective already corrected); only the grau/scan filter is applied.
- **Filename format**: `YYYY-MM-DD_Betrag_Zahlungsart.pdf` or `YYYY-MM-DD_Zahlungsart.pdf`

## Gotchas

- `pnpm run build:android` (not `build`) for the APK — regular build needs `PORT`/`BASE_PATH` env vars.
- After changing source files, always re-run `build:android` then `cap sync android` before building the APK.
- `@capacitor-mlkit/text-recognition` does NOT exist on npm. Use Tesseract.js instead.
- `CapacitorConfig` is imported from `@capacitor/cli`, not `@capacitor/core`.
- Android .gitignore (auto-generated by Capacitor) excludes `app/src/main/assets/public` — regenerate via `cap sync`.
